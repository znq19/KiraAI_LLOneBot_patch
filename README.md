# KiraAI · LLOneBot 兼容补丁

让 KiraAI 能完整读取 LLOneBot 的语音和合并转发消息（含嵌套），仅需替换一个文件。

**本补丁基于 [LLOneBot / LuckyLilliaBot](https://github.com/LLOneBot/LuckyLilliaBot) 修改，遵循 GPL 2.0 许可证。**

> 全程只覆盖一个文件，不会动 KiraAI 本体。

最佳适用版本：**LLBot 7.12.14**（怎么看版本见下文，不是非这个版本不可，只是我基于它改的）

更新日期：2026-07-17

修改：@znq19

---

## 这补丁干了啥？

简单说：你用 LLBot 连着 QQ，KiraAI 当机器人时之前有几个毛病：

- 群里发的**合并转发消息**，KiraAI 读不出来，或者只显示个 `[转发消息]` 就没了
- **转发里面还套着转发**（嵌套），直接空白或者报错
- 别人发的**语音**，KiraAI 听不到

这个补丁就是修这些的。改完以后：
- 合并转发能正常看了，里面每条消息 KiraAI 都能读到
- 套娃转发也能一层层展开
- 语音能识别了

---

## 第一步：确认你的 LLBot 版本

打开 LLBot 的安装目录，找到这个文件：

```
bin\llbot\package.json
```

用记事本打开，搜 `"version"`，看是不是 `7.12.14`。

- 是 → 无条件继续往下
- 不是 → 也无所谓，但不保证一定好使，试试就知道

---

## 第二步：替换文件

> ⚠️ 操作之前，**先把 LLBot 彻底关掉**！
> 关了窗口还不够，右下角托盘里如果有图标，右键退出。

1. 在 `bin\llbot\` 目录下，找到 `llbot.js`
2. 把它**复制一份**改名 `llbot.js.bak`（备份，以防万一）
3. 用补丁里改好的 `llbot.js` **覆盖**原来的那个

就这三步，完事。

---

## 第三步：启动测试

1. 先打开 **LLBot**，登录 QQ，确认连上了
2. 再打开 **KiraAI**
3. 找个群，**发一条新的合并转发或语音**来测（旧消息可能还是老缓存，测不出来）

怎么算成功了？
- KiraAI 日志里能看到合并转发里的每一条消息内容，不是只显示个 `[转发消息]`
- 套娃转发能看到层级展开，不会空白
- 发语音不会报错了

## 想自己调调？（可选）

默认配置已调整好最真实能让 AI 看全的状态，一般不用动。但如果你想改：

用记事本打开 `bin\llbot\llbot.js`，搜：

```
KIRA_FORWARD_MAX_NEST_DEPTH
```

找到这两行（在文件靠顶部的位置）：

```js
// ========== KiraAI 合并转发开关（用户只改下面两行即可）==========
// 0=最外层完整保留，内层合并转发卡片不展开（带预览占位）
// 1~4=允许展开的嵌套层数（越大越深，上限4）
var KIRA_FORWARD_MAX_NEST_DEPTH = 4;
// true=嵌套内容保留真实图片/语音/视频；false=嵌套内媒体一律 [图片]/[语音]/[视频] 文本
var KIRA_FORWARD_NESTED_MEDIA = true;
// =================================================================
```

**只改这两个 `var` 后面的值就行**，别动其他地方。

### 开关 1：`KIRA_FORWARD_MAX_NEST_DEPTH`

| 值 | 效果 |
|----|------|
| `0` | 只看最外层，里面的合并转发不展开，显示 `[嵌套转发] 预览` |
| `1` ~ `4` | 允许展开对应层数，数字越大越深，最大 4 |

### 开关 2：`KIRA_FORWARD_NESTED_MEDIA`

这个开关现在支持**三种模式**（v12 新增）：

| 值 | 效果 |
|----|------|
| `true` | 全部保留真图真语音 |
| `false` | **仅嵌套层**改文字占位，最外层还是真图 |
| `"none"` | **全部**（最外层 + 嵌套层）都改文字占位 |

> ⚠️ **`"none"` 必须带引号**，写成 `"none"`，不是 `none`！不带引号 JS 会直接报错。

常见改法：

- **只想看最外层，里面不管**：
  ```js
  var KIRA_FORWARD_MAX_NEST_DEPTH = 0;
  ```

- **要展开但嵌套里的图片不要**（最外层还是真图）：
  ```js
  var KIRA_FORWARD_NESTED_MEDIA = false;
  ```

- **全部不要图片，纯文字最省 token**：
  ```js
  var KIRA_FORWARD_NESTED_MEDIA = "none";
  ```

改完记得**关掉 LLBot 再重开**才生效。

> 不用去改什么 `config_*.json`，开关**只看 `llbot.js` 里这两个 `var` 常量**。以前如果在 JSON 里写过旧变量名可以忽略或删掉，避免搞混。

## 注意事项

1. **升级 LLBot 后补丁会没掉**——官方更新会覆盖 `llbot.js`，得重新打一次补丁
2. **一定要彻底退出再改文件**——托盘里挂着就是白改
3. **测试要发新消息**——旧消息可能还是缓存，看起来像没修好
4. **这个补丁没动 KiraAI 任何代码**——所有改动都在 LLBot 这边
5. 语音这块：内置 ffmpeg 精简版不支持 mp3，补丁会转成 **wav** 再给 KiraAI

## 嵌套转发长啥样？（给 AI 看的格式）

改完以后，KiraAI 看到的嵌套转发大概是这样：

```
┌─嵌套转发#1
  周武: 456
  周武:
    ┌─嵌套转发#2
      周武: 1
      周武: 2
      周武: 3
    └─嵌套转发#2结束
└─嵌套转发#1结束
```

层级清楚，谁说了啥一目了然，图片会跟在对应消息后面，不会全堆在末尾。

如果把 `MAX_NEST_DEPTH` 设成 `0`，内层合并转发就只会显示一行：

```
[嵌套转发]
```

不会展开整棵树。
